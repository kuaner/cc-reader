# Surgery Mode（手術モード）設計書

## 概要

Claude Codeのコンテキスト（JSONLファイル）を直接編集する機能。
不要なやり取りを削除・圧縮して、トークン消費を最適化する。

## 背景・課題

- Claude Codeは毎ターン全メッセージをAPIに送信
- 長いセッションはコンテキスト上限に達し自動要約される
- 自動要約は大雑把で重要な文脈が失われることがある
- 間違った指示や修正のやり取りがそのまま残りトークンを浪費

## 解決策

JSONLを直接編集し、コンテキストを「外科手術」的に整形する。

## 操作

### 1. 削除

選択したメッセージをJSONLから完全に削除。

**使用例:**
- 間違った指示とその応答を丸ごと削除
- 試行錯誤のやり取りを削除

**注意:**
- parentUuidのチェーンが切れる可能性あり → 警告表示

### 2. 要約に置換

選択した複数メッセージを1つの要約メッセージに置換。

**処理フロー:**
1. 選択されたメッセージ群をClaudeに送信
2. 「以下のやり取りを1-2文で要約してください」
3. 生成された要約で置換

**使用例:**
- 長いデバッグのやり取り → 「認証のバグを修正した」
- DB選択の議論 → 「PostgresからMySQLに変更した」

### 3. 編集

単一メッセージの内容を直接編集。

**編集可能な部分:**
- `content[].text` - テキスト内容
- `content[].thinking` - 思考プロセス（削除推奨）
- `content[].tool_result.content` - ツール実行結果（圧縮推奨）

**使用例:**
- 間違った指示を修正
- tool_resultの巨大なファイル内容を「ファイルX読み込み済み」に置換
- thinkingを削除してトークン節約

### 4. thinkingを一括削除

アシスタントメッセージのthinkingブロックを一括削除するユーティリティ。
thinkingは再実行時に不要で、トークン節約効果が高い。

## データ構造

### JSONLの行構造（RawMessageData）

```json
{
  "type": "user",
  "uuid": "xxx",
  "parentUuid": "yyy",
  "sessionId": "zzz",
  "timestamp": "2026-01-28T10:30:45.123Z",
  "cwd": "/path/to/project",
  "gitBranch": "main",
  "slug": "session-name",
  "message": {
    "role": "user",
    "content": "テキスト内容",
    "model": null
  }
}
```

### アシスタントメッセージのcontent構造

```json
{
  "message": {
    "role": "assistant",
    "content": [
      {"type": "thinking", "thinking": "思考プロセス..."},
      {"type": "text", "text": "応答テキスト..."},
      {"type": "tool_use", "id": "xxx", "name": "Read", "input": {...}},
      {"type": "tool_result", "tool_use_id": "xxx", "content": "結果..."}
    ],
    "model": "claude-opus-4-5-20251101"
  }
}
```

### トークン消費の内訳（典型例）

| ブロックタイプ | 平均トークン数 | 備考 |
|---------------|---------------|------|
| thinking | 500-5,000 | 削除推奨、再実行時不要 |
| text | 100-2,000 | 本文、基本的に保持 |
| tool_use | 50-200 | 小さい、保持推奨 |
| tool_result | 100-50,000 | 最大、ファイル内容で膨張 |

## 実装計画

### Phase 1: 基盤

1. **SurgeryModeManager** - 手術モードの状態管理
   - 選択状態
   - 変更のプレビュー
   - Undo/Redo

2. **TokenEstimator** - トークン数推定
   - 文字数ベースの簡易推定（4文字≒1トークン）
   - または tiktoken 相当のライブラリ

3. **JSONLWriter** - JSONLファイルへの書き込み
   - 元ファイルのバックアップ
   - アトミックな書き込み

### Phase 2: UI

4. **SurgeryOverlayView** - 手術モードのオーバーレイUI
   - トークンバー
   - アクションボタン
   - 削減予定表示

5. **MessageRow拡張** - チェックボックス・トークン数表示
   - 手術モード時のみ表示
   - 選択時の半透明表示

### Phase 3: 操作

6. **削除機能**
7. **編集機能** - インラインエディタ
8. **要約置換機能** - Claude API連携

### Phase 4: 安全性

9. **バックアップ・リストア**
10. **parentUuid整合性チェック**
11. **変更プレビュー（diff表示）**

## 注意事項

### 安全性

- **元ファイルのバックアップ必須** - 手術前に自動バックアップ
- **Claude Code実行中は編集禁止** - ファイルロック or 警告
- **parentUuidの整合性** - チェーン切れを検出して警告

### 制限事項

- Claude Codeがセッション再開時にJSONLを読み直す前提
- 編集後、Claude Codeの再起動またはセッション再読み込みが必要な可能性
